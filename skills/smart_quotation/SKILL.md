# Smart Quotation - 智能报价核心 Skill

## 核心能力

1. **报价入口分流器**：所有现有和后续产品统一先判断 `single_item` / `requirement_p0` / `unclear`
2. **价格计算引擎**：基于产品库 + 价格策略，100% 确定性计算
3. **规则校验引擎**：7 项自动检测（产品存在、价格完整、库存充足等）
4. **分流派单引擎**：标准型/方案型/非标型/高风险 四类路由
5. **文档生成引擎**：生成内部版、客户版报价单（Markdown/PDF）
6. **审批流程引擎**：状态机驱动的多级审批 + 通知推送

## 报价入口分流器（全产品通用）

所有产品报价在进入产品专属价格引擎前，必须先调用统一分类器：

```python
classify_quotation_entry(message, payload)
```

分类结果：

| flow_type | 场景 | 处理规则 |
|---|---|---|
| `single_item` | 单品查价、库存、价格参考、明确产品编码/配置的轻量询价 | 直接走快速报价/参考报价，不进入正式客户报价流程 |
| `requirement_p0` | 客户/项目/方案/P0/正式报价/客户报价单/交付/预算/附件/多产品组合 | 必须先创建客户需求表三，再进入需求卡、分流、规则校验、审批和客户版输出 |
| `unclear` | 无法判断是单品查价还是正式客户需求报价 | 先追问用户确认报价入口 |

强约束：

- 当前和后续所有产品 Skill 必须复用该分类器，不允许各产品自行绕过入口判断。
- 只要命中客户、项目、方案、正式输出或 P0 信号，即使同时包含“价格/报价”，也按 `requirement_p0` 处理。
- `single_item` 只能输出内部快速查价/参考报价，不代表正式客户报价。
- 无影云电脑单品查询、快速查价、项目需求报价、正式报价在对外对话中统一使用无影报价表：云电脑资源表固定列为“序号 / 产品名称 / 时长类型 / 产品/配置 / 数量 / 月价 / 总价（月） / 一年总价”；无影终端表固定列为“序号 / 产品名称 / 产品/配置 / 数量 / 单价 / 总价”；最后输出云电脑资源月总价、云电脑资源一年总价、无影终端总价、首月合计、一年合计。月价/单价必须来自无影价格引擎/报价库；一年总价必须按“数量 × 月价 × 12”重算，不得读取旧表“一年价”列；“无影魔方/无影魔法 AS05”必须识别为无影终端硬件并进入终端表。不得再使用“价格场景 / 一年价 / 计价单位 / 价格策略”旧表头。
- 正式客户报价不得绕过客户需求表三、人工确认、规则校验和审批。
- **精准匹配原则**：凡是产品/配置/时长/数量/终端型号无法在报价库中精准定位到唯一价格的，必须进一步追问确认；不得根据模糊匹配、相似产品、历史印象或模型推断输出价格。查不到精准价格时，回复应说明缺少哪些信息，并给出可选配置/时长/终端候选。

- 单品报价解决的是“查价格”，只能输出内部参考报价；项目需求报价解决的是“正式客户报价流程”，必须先进入客户需求表三，再进入需求卡、分流、规则校验、人工确认和客户版输出。

```json
{
  "customer_company": "XX教育科技有限公司",
  "customer_level": "standard|vip|strategic|new",

  "budget_range": "20000-25000",
  "custom_requirements": "",
  "communication_needed": true
}
```
