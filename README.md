# HubAI 企业级 AI 集成平台

## 项目简介

HubAI 是面向企业级场景的 AI 集成平台，专注于**工作流程自动化**、**智能报价**和**知识管理**三大核心功能。通过 OpenClaw Skills 与 MCP 协议，实现与钉钉生态的深度整合。

## 核心价值

| 价值维度 | 核心内容 |
|---|---|
| **架构** | 认知与执行分离（大模型调度 → Skill 执行） |
| **入口** | 统一报价入口分类器（单品 / 需求 P0 / 不明确） |
| **流程** | 客户需求表三必经，历史项目仅参考 |
| **落地** | 无影云电脑等产品线的完整报价流程 |
| **记忆** | 长期记忆固化 + 知识沉淀体系 |
| **运维** | 每日定时归档 + 记忆更新 + 钉钉通知推送 |

## 技术栈

| 层级 | 核心技术 |
|---|---|
| **用户接口** | 钉钉对话、网页管理后台（Vue3）、飞书/企微集成（待开发） |
| **统一接入** | OpenClaw Gateway、MCP 协议、FastAPI |
| **业务执行** | Python Skills（hubai_base / smart_quotation）、OpenClaw Runtime |
| **数据存储** | SQLite + FTS5（全文搜索）、JSONL（日志） |
| **内容解析** | Jina Reader、OpenAI GPT-4o-mini（轻量级解析） |
| **定时任务** | Cron（OpenClaw 内置）、systemd 服务 |
| **消息通知** | 钉钉（dingtalk-connector）、飞书/企微（待开发） |
| **部署方式** | Docker Compose、本地单节点部署 |

## 部署与运行

### 前置条件

| 依赖 | 版本 | 说明 |
|---|---|---|
| **Python** | 3.10+ | |
| **OpenClaw** | 2.0+ | |
| **systemd** | 245+ | Linux 用户级服务 |

### 启动命令

```bash
# 启动 DingTalk OpenClaw Bridge
sudo systemctl --user start dingtalk-openclaw-bridge.service

# 检查服务状态
sudo systemctl --user status dingtalk-openclaw-bridge.service

# 查看运行日志
journalctl --user -u dingtalk-openclaw-bridge.service

# 启动知识库服务（FastAPI）
cd /home/xhb-szwl/.openclaw/workspace/Knowledge-Library-MVP
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 定时任务

| 任务名 | 执行时间 | 功能 |
|---|---|---|
| **hubai_daily_archive_memory_update** | 每日 23:00 | 变更检测 + 归档 + 记忆更新 |
| **hubai_daily_archive_dingtalk_notification** | 每日 08:40 | 前一日执行摘要推送到钉钉 |
| **hubai_weekly_archive_summarize** | 每周一 10:00 | 上周变更统计 + 周度摘要 |

## 安全与合规

### 数据安全

| 措施 | 说明 |
|---|---|
| **本地存储** | SQLite 数据库，不存储外部敏感信息 |
| **敏感字段** | `.env` 文件存储（`.gitignore` 保护） |
| **操作审计** | 所有定时任务操作记录到 `delivery_queue_entries` |

| 机制 | 说明 |
