# product-wuying-pc - 无影云电脑报价 Skill

## 核心能力

1. **需求表管理** - 12个字段的需求表，覆盖基础信息、配置需求、用户需求、部署需求、商务需求
2. **对话话术** - 5套话术模板，支持欢迎、追问、汇总、报价完成等场景
3. **价格计算** - 云产品资源配置来自报价数据“国内region计算规格名称”，硬件设备来自 sheet“C0-终端描述汇总”，支持云盘附加费用

## 配置文件

- `config/requirement.json` - 需求表配置
- `config/dialogue.json` - 对话话术配置

## 脚本模块

- `scripts/requirement_handler.py` - 需求处理器（继承 BaseRequirementHandler）
- `scripts/dialogue_handler.py` - 对话处理器（继承 BaseDialogueHandler）
- `scripts/price_engine.py` - 价格引擎

## 价格体系

- 报价模式：支持【单品查询】和【方案报价】两种模式。
  - 单品查询：按云产品资源配置/硬件设备/产品名称直接查价；对话输出必须使用统一无影报价格式，云电脑资源表展示“月价 / 总价（月） / 一年总价”，无影终端表单独展示“单价 / 总价”，不得使用旧的“价格场景 / 一年价 / 计价单位 / 价格策略”表头，不展开 2-6 年价格。
  - 方案报价：必须先补齐需求清单必要项后才能报价。
- **单品报价 / 项目需求报价边界**：单品报价解决的是“查价格”，只能作为内部参考报价；项目需求报价解决的是“正式客户报价流程”，客户/项目/方案/P0/正式报价必须先进入客户需求表三，再进入需求卡、分流、规则校验、人工确认和客户版输出。
- **单品查询 / 项目需求报价统一输出格式硬规则**：
  - 无影云电脑资源表固定列为：序号 / 产品名称 / 时长类型 / 产品/配置 / 数量 / 月价 / 总价（月） / 一年总价。
  - 计算口径：月价必须来自无影价格引擎/报价数据库，不允许在钉钉出口或模型回复中自行猜测；总价（月）= 数量 × 月价；一年总价 = 数量 × 月价 × 12。注意：一年总价不得读取旧表中的“一年价”列，必须按公式重算。
  - 无影终端单独报价，固定列为：序号 / 产品名称 / 产品/配置 / 数量 / 单价 / 总价。
  - 最后必须基于“无影云电脑资源报价”和“无影终端报价”给出汇总：云电脑资源月总价、云电脑资源一年总价、无影终端总价、首月合计、一年合计。
  - 单品查询和需求/项目报价都使用上述统一格式；单品查询仍为内部参考报价，项目/正式报价仍必须先进入客户需求表三并经人工确认。
  - 如果未选择无影终端，终端表显示“未选择/不需要”，金额为 0。
  - 只展示【月价】、【总价（月）】、【一年总价】；不得在对话中展开 2年、3年、4年、5年、6年价格；输出优先使用 Markdown 表格。
- 价格策略：按报价数据源原价报价，不做折扣。
- **精准匹配原则**：凡是产品/配置/时长/数量/终端型号无法在报价库中精准定位到唯一价格的，必须进一步追问确认；不得根据模糊匹配、相似产品、历史印象或模型推断输出价格。查不到精准价格时，回复应说明缺少哪些信息，并给出可选配置/时长/终端候选。

- 云产品资源配置：对应报价数据源字段“国内region计算规格名称”。
- 休眠方式：方案报价前必须询问并确认；选项从云产品资源配置字段中文括号“（）”内自动提取，目前为“支持24小时开机”“无强制休眠策略”。
- 硬件设备：对应报价数据源 sheet“C0-终端描述汇总”的“终端规格”。“无影魔方/无影魔法 AS05”统一识别为无影终端硬件“无影魔方AS05（推荐）”，必须进入无影终端报价表，不得作为云电脑资源行或月费/年费合计行。
- 使用场景固定为：安全办公、教育培训、研发设计、数字人&媒体运营、AI agents、其他。不得私自添加或修改，如需变更必须先确认。
