### 2.2 智能追问逻辑（采用字段优先级与缺失检测驱动）

```python
def next_question(collected, context):
    """
    智能决定下一个问题
    策略：优先级 > 缺失检测 > 关联字段合并 > 业务场景驱动
    """
    # Step 1: 识别所有缺失字段
    missing = identify_missing_fields(collected)
    
    if not missing:
        return None, "信息收集完成，开始生成需求卡！"
    
    # Step 2: 按优先级排序
    sorted_fields = sort_by_priority(missing)
    next_field = sorted_fields[0]
    
    # Step 3: 关联字段合并提问（优化交互体验）
    # 姓名+电话合并
    if next_field == 'customer_name' and 'customer_phone' in missing:
        return "方便告诉我联系人姓名和电话吗？"
    
    # 业务场景+产品合并（如果同时缺失且P1/P0相邻）
    if next_field == 'business_scenario' and 'products' in missing:
        return "请描述一下客户的业务场景，以及需要哪些产品、数量多少？"
    
    # 预算+技术方案合并（如同时缺失）
    if next_field == 'budget_range' and 'communication_needed' in missing:
        return "客户预算范围是多少？是否需要沟通交流和技术方案？"
    
    # Step 4: 业务场景驱动的智能追问
    # 如果已经有业务场景，可根据场景调整问题
    if 'business_scenario' in collected:
        scenario = collected['business_scenario']
        if '教育' in scenario or '学校' in scenario:
            if 'student_count' in missing:
                return "学生规模大概多少人？"
        if '企业' in scenario or '办公' in scenario:
            if 'employee_count' in missing:
                return "员工规模大概多少人？"
    
    # Step 5: 默认按优先级提问
    return question_templates[next_field]
```

### 2.3 字段缺失检测算法

```python
def identify_missing_fields(collected):
    """
    基于规则的字段缺失检测
    返回：按优先级排序的缺失字段列表
    """
    required_fields = {
        'P0': ['customer_company', ('customer_name', 'customer_phone'), ('products', 'quantity')],
        'P1': ['business_scenario', 'budget_range'],
        'P2': ['communication_needed'],
        'P3': ['custom_requirements', 'competitor_info', 'payment_terms']
    }
    
    missing = []
    for priority in ['P0', 'P1', 'P2', 'P3']:
        for field in required_fields[priority]:
            # 处理元组形式的组合字段（需同时满足）
            if isinstance(field, tuple):
                if not all(f in collected and collected[f] for f in field):
                    missing.append((priority, field[0] if isinstance(field[0], str) else field))
            else:
                if field not in collected or not collected[field]:
                    missing.append((priority, field))
    
    return [f[1] for f in missing]

## 三、对话流程节点设计

### 3.1 完整流程图

```
[开始]
  ↓
[节点1：欢迎 + 信息收集]
  ↓ 收集完成
[节点2：生成需求卡]
  ↓ 展示完整性评分
[节点3：缺口确认]
  ↓ 用户补全或确认跳过
[节点4：需求分流]
  ↓ 标准型/方案型/非标型/高风险
[节点5：历史案例参考]
  ↓ 展示相似案例（仅供参考）
[节点6：输出类型确认]
  ↓ 方案/报价单/两者都要
[节点7：生成报价草稿]
  ↓ 展示内部版 + 风险提示
[节点8：审批/修改]
  ↓ 审批通过或用户确认
[节点9：生成客户版]
  ↓ Markdown/PDF 格式
[节点10：发送确认]
  ↓ 用户确认发送给客户
[节点11：回写知识库]
  ↓ 完成归档
[结束]
```
