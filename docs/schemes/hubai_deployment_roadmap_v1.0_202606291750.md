## 五、风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| OpenClaw Skill 调用机制不熟悉 | 集成延迟 | 先做最小可行验证（hello world），再逐步复杂 |
| 产品价格数据缺失 | 报价不准确 | 先使用示例价格占位，后续接入真实 ERP |
| 钉钉审批 API 对接复杂 | 审批流延期 | 先做"人工确认"降级方案，后续再自动化 |
| 多产品需求表差异大 | 复用度低 | 提取通用字段到基类，产品只保留差异化字段 |
